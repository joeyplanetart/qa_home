# 使用文档

## 访问方式

- **本地：** http://localhost:8765（运行 `./scripts/run.sh` 后）
- **线上：** 部署在 Vercel 后的域名（见 [部署文档](DEPLOY.md)）

> 必须通过后端服务访问，不能直接用浏览器打开 `index.html` 文件。

## 项目 Tab

### 浏览项目

项目按业务线分组：

- 🏢 PlanetArt 内部（ADMIN、Devtools、Cordial 等）
- ☕ CafePress
- 🎴 卡片 & 印刷
- 📱 手机壳
- 🎁 礼品 & 生活

每张项目卡片显示：

- 项目名称、ID、域名
- 环境标签（prod / pre / staging）
- 备忘录 / 流程 / 片段 / **自动化用例** 数量统计（🤖 图标）
- 健康检查状态（绿 / 黄 / 红）

自动化用例数来自 `automation/suites/*/meta.json` 中的 `projectId` 关联，对应用例文件中的 `test_*` 函数数量。

### 筛选与搜索

- **顶栏项目下拉：** 选中后，备忘录 / 流程 / 片段 Tab 只显示该项目的内容
- **项目 Tab 搜索框：** 按名称、域名、ID 过滤
- **分组按钮：** 快速切换业务线

### 健康检查

点击 **🩺 健康检查** 按钮，后端会探测各项目站点 HTTP 可达性：

- 🟢 正常（2xx / 3xx）
- 🟡 异常响应
- 🔴 不可达或超时

默认每 5 分钟自动刷新一次。

### 快捷操作

项目卡片底部三个按钮：

| 按钮 | 作用 |
|------|------|
| 备忘录 | 筛选该项目并跳转到备忘录 Tab |
| 流程 | 筛选该项目并跳转到操作流程 Tab |
| 访问站点 | 新窗口打开项目域名 |

## 备忘录

用于记录测试环境地址、Bug 排查要点、API 说明等。

### 创建 / 编辑

1. 点击右上角 **📝** 或点击已有卡片
2. 填写标题、内容
3. 选择所属项目（留空 = 通用/跨项目）
4. 选择分类、颜色，可勾选置顶
5. 保存

### 分类筛选

顶部分类标签：全部 / 通用 / API / 数据库 / 环境 / Bug / 其他

### 搜索

全局搜索框可搜备忘录标题和内容，点击结果直接打开编辑。

## 操作流程

记录步骤化的业务操作，例如：

- ADMIN 订单状态变更
- STI 新用户注册冒烟
- 生产环境发布流程

### 创建

1. 点击右上角 **⚙️**
2. 填写标题、描述
3. 步骤每行一个
4. 标签用逗号分隔（如 `deploy, production`）
5. 选择所属项目

### 查看

点击流程标题展开 / 收起步骤详情。

## UI 自动化

入口：首页顶部 **🤖 UI 自动化**，或直接访问 `/automation`。

> UI 自动化仅在本地运行（需安装 Playwright 浏览器），Vercel 部署环境不支持执行测试。

### 前置条件

```bash
./scripts/setup.sh
./scripts/install-playwright.sh   # 安装 Chromium
./scripts/run.sh
```

### 界面说明

```
┌──────────────┬────────────────────────────────────────────┐
│  套件列表     │  Tab：测试用例 | 执行结果 | 历史 | 日志 | 报告 │
│  运行配置     │  用例列表 + 源码查看 / 运行结果详情           │
│  运行按钮     │                                            │
└──────────────┴────────────────────────────────────────────┘
```

- **测试用例 Tab**（默认）：按文件分组浏览用例，支持勾选、查看源码（Prism 高亮 + 行号）
- **执行结果 Tab**：本次运行的通过/失败统计、用例详情、测试数据（账号、site_id 等）
- **历史记录 / 日志 / 报告**：查看历次运行、终端输出、HTML 报告

### 运行方式

| 操作 | 说明 |
|------|------|
| 运行选中 (N/M) | 仅运行勾选的 N 个用例（侧边栏与用例列表按钮行为一致） |
| 运行套件 (M) | 无勾选时，运行当前套件全部 M 个用例 |
| 运行全部 | 运行 `automation/suites` 下所有套件 |
| 文件 ▶ | 运行单个测试文件 |
| 用例 ▶ | 运行单个测试函数 |

### 运行配置

左侧面板可配置（自动保存到浏览器 localStorage）：

- 有头 / 无头模式、浏览器类型
- 视口尺寸、设备模拟、慢动作、超时
- 语言、录像、Trace

### 用例标记

在测试代码中使用 pytest marker：

```python
@pytest.mark.selected   # 加载时默认勾选
@pytest.mark.smoke      # 冒烟测试（界面显示「冒烟」徽章）
def test_example(page):
    ...
```

标记定义见 `automation/pytest.ini`。

### 测试数据（Artifacts）

用例中通过 `test_data` fixture 记录的数据（如注册邮箱、customer_id、site_id）会写入 `reports/{runId}/artifacts/`，并在执行结果中展示。

### 套件与项目关联

每个套件目录下的 `meta.json` 示例：

```json
{
  "name": "CafePress E2E",
  "description": "CafePress 美国主站 E2E",
  "projectId": 170
}
```

`projectId` 对应 `projects.js` 中的项目 ID，用于首页卡片 🤖 用例数统计。

### 目录结构

```
automation/suites/{suite_id}/
├── meta.json           # 套件元数据
├── conftest.py         # 套件级 fixture（可选）
├── pages/              # Page Object
└── test_*.py           # 测试用例
```

## 数据说明

- 所有内容保存在数据库中（Turso 或本地 SQLite），刷新页面不丢失
- 项目列表配置在 `server/static/assets/js/projects.js`，修改后需刷新页面
- UI 自动化运行记录存数据库，报告/截图/artifacts 存 `reports/` 目录（本地）
- 删除操作无确认弹框，执行后通过右下角 toast 提示结果

## 常见问题

**Q: 页面提示「数据加载失败」？**  
A: 确认 `./scripts/run.sh` 已启动，且端口未被占用。

**Q: 项目 Tab 空白？**  
A: 硬刷新（Cmd+Shift+R），或重启服务。

**Q: 如何新增一个 PlanetArt 项目？**  
A: 编辑 `server/static/assets/js/projects.js` 中的 `PROJECTS` 数组，按现有格式添加 `id`、`short_name`、`domain`、`group` 等字段。

**Q: 通用内容和项目内容有什么区别？**  
A: 创建时可不选项目（标记为「通用」），这类内容不属于任何单一项目；选中项目筛选后只显示该项目专属内容。

**Q: UI 自动化提示需要安装 Playwright？**  
A: 运行 `./scripts/install-playwright.sh` 安装 Chromium 浏览器。

**Q: 如何新增一个自动化测试套件？**  
A: 在 `automation/suites/` 下新建目录，添加 `meta.json` 和 `test_*.py`；在 `meta.json` 中设置 `projectId` 关联首页项目。
